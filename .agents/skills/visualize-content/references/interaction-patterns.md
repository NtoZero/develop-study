# 학습 주제별 인터랙션 패턴

기술 아티클의 특정 문단이 설명하는 인과관계를 가장 적은 조작으로 드러내는 inline figure 패턴을 선택하라. 독자가 먼저 본문으로 질문과 변수를 이해한 뒤 figure를 조작하게 하며, 여러 패턴을 하나의 대시보드에 합치지 마라.

## 선택 원칙

- 학습자가 먼저 결과를 예측하고, 조작한 뒤, 실제 결과와 이유를 확인하게 하라.
- 장식적 움직임이 아니라 상태, 순서, 동시성, 병목 또는 트레이드오프를 보여라.
- 실제 시스템을 재현하지 못하는 계산은 `교육용 모델`로 표시하고 가정과 한계를 설명하라.
- 무작위 결과가 학습에 필수적이지 않으면 결정론적으로 구현해 재현과 비교가 가능하게 하라.
- 퀴즈, 카드 뒤집기, 탭 전환만으로 도메인 인터랙션을 대체하지 마라.

## 패턴 매핑

| 주제 신호 | 주 인터랙션 | 의미 있는 조작 | 관찰할 피드백 |
| --- | --- | --- | --- |
| 순차 처리, 파이프라인, 데이터 플로우 | 단계 실행기와 흐름 강조 | 입력, 단계 이동, 재생 속도 | 현재 데이터, 변환 전후, 병목 |
| 상태 머신, 수명 주기, 프로토콜 | 상태 전이 샌드박스 | 이벤트 발생, 조건 변경 | 현재 상태, 허용·거부 전이, 이유 |
| 알고리즘, 자료구조 | 실행 추적기 | 입력 데이터, 실행·한 단계 이동 | 비교·이동 횟수, 중간 구조, 결과 |
| 네트워크, 분산 시스템 | 레인 기반 타임라인 | 지연, 순서, 손실, 장애 주입 | 메시지 위치, 타임아웃, 재시도, 합의 상태 |
| 데이터베이스, 쿼리 | 실행 계획 또는 페이지 탐색기 | 조건, 인덱스 유무, 데이터 분포 | 접근 경로, 읽은 페이지, 예상 비용 |
| 트랜잭션, 동시성 | 인터리빙 스케줄러 | 실행 순서, 격리 수준, 잠금 | 읽은 값, 대기 관계, 이상 현상 |
| 브라우저 렌더링, 이벤트 루프 | 큐·프레임 타임라인 | 태스크 추가, 우선순위, 작업 시간 | 스택·큐 상태, 프레임 지연, 페인트 시점 |
| 백엔드 ORM, 데이터 로딩 | 호출·쿼리 추적기 | 로딩 전략, 연관 접근, 배치 크기 | 발생 SQL, 왕복 수, 객체 상태 |
| 인프라, 부하 테스트, 관측 가능성 | 본문 폭을 넘기는 시간축 figure | VU/도착률 전환, 지연 또는 VU 상한 중 문맥상 한두 값 | 시작률, 동시성, 누락의 인과관계 |
| 비교, 선택 기준 | 조건 기반 비교 실험 | 워크로드와 제약 조건 | 대안별 이점·비용과 선택 근거 |
| 계층, 아키텍처, 소유 관계 | 드릴다운 구조 지도 | 노드 선택, 계층 펼치기 | 책임, 입력·출력, 상하위 연결 |

## 대표적인 특화 예시

### Spring Data JPA

엔티티 관계 그래프와 SQL 추적기를 결합하라. 지연 로딩, fetch join, entity graph 또는 배치 크기를 바꾸면 어떤 접근이 어떤 SQL과 왕복을 만드는지 단계별로 보여라. 데이터 규모와 영속성 컨텍스트 조건을 명시하고 단순히 `쿼리가 적으면 항상 좋다`고 결론 내리지 마라.

### k6

closed/open model을 설명한 문단 직후에는 지연 하나를 바꾸며 처리율 또는 필요한 VU가 어떻게 달라지는지 보여주는 작은 시간축 figure를 둬라. threshold 절에는 응답 표본과 p95 기준선만 다루는 별도 figure를 둔다. 모든 stages, VU, rate, error, threshold와 KPI를 하나의 control deck에 모으지 마라. 실제 측정값이 아닌 예시는 시뮬레이션임을 캡션에 표시하라.

### B-tree 인덱스

키 분포와 페이지 크기를 바꾸며 탐색 또는 삽입을 한 단계씩 실행하게 하라. 현재 페이지, 비교한 키, 다음 포인터와 split 결과를 보여라. 실제 DB의 구현 세부사항과 단순화된 B-tree 모델의 차이를 밝히라.

### JavaScript 이벤트 루프

호출 스택, task queue, microtask queue와 렌더링 기회를 나란히 배치하라. 코드 조각을 실행하기 전에 로그 순서를 예측하게 하고 각 단계에서 어느 큐가 선택되는지 설명하라.

### 네트워크 프로토콜

클라이언트와 서버 레인을 사용해 패킷, 지연, 재전송과 타임아웃을 시간순으로 보여라. 패킷 손실이나 지연을 조작할 수 있게 하되 프로토콜 버전과 생략한 세부 동작을 표시하라.

## 패턴 품질 검사

- 조작이 단순 UI 상태가 아니라 학습 대상의 상태를 바꾸는가?
- 사용자가 조작 전 결과를 예측할 기회가 있는가?
- 변화한 값과 변화하지 않은 불변 조건이 모두 보이는가?
- 실패 결과가 단순 오류 문구가 아니라 원인과 복구 방법을 설명하는가?
- 한 번 실행한 과정을 초기화하고 동일하게 재현할 수 있는가?
- 애니메이션을 끄더라도 단계와 인과관계를 이해할 수 있는가?
